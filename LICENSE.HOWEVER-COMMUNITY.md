# HoweverUnsloth 社区协作协议（补充说明）

本文档是对仓库工程化协作方式的补充约定，不替代原仓库中已有的开源许可证文本。

## 1. 适用范围

- 适用于本仓库新增的私有化配置、脚本、文档与品牌化素材。
- 对原有上游代码继续遵循其对应许可证要求。

## 2. 署名与保留

- 二次分发时应保留原作者署名、许可证声明和变更记录。
- 对新增文件应注明维护者与变更日期，便于审计与回溯。

## 3. 商业与内部使用

- 允许企业内部使用、二次开发与部署。
- 涉及对外商用分发时，使用方需自行完成合规审查（包括模型、数据与内容安全）。

## 4. 安全责任

- 使用方应自行维护 API Key、数据库口令和访问控制。
- 因密钥泄漏、配置不当导致的数据风险由部署方承担。

## 5. 贡献约定

- 贡献代码需遵循最小改动原则，避免破坏上游兼容性。
- 建议在 PR 中包含：改动范围、验证步骤、回滚方案。

## 6. 免责声明

- 本项目按“现状”提供，不承诺特定商业结果。
- 维护者不对因部署环境、第三方依赖或外部服务不可用导致的损失负责。
